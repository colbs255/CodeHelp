Classes and Objects

Now you can be introduced to Object Oriented Programming (OOP). This is what Java is all about, and just about every program you create in your career will utilize concepts of OOP. Object Oriented Programming is about creating your own types of data (objects) and manipulating them in your own specified way. This makes modelling things in the real world much easier and organized. For example, let's say you wanted to make a video game with dogs in it. You would make your own Dog object that can bark, eat, or run, and it will also have its own characteristics such as hair color, height, and weight.

Objects

You have used objects before. The scanner that you use for input is an object. Objects are designed by classes. The class gives the layout of what of kind of data an object contains (global variables) and how the object can manipulate the data (methods). For example, a dog object would have a global string variable called name to hold the name of the dog, a global string variable for the breed, and a global int for the age. Your Dog class would have all of these variables (or more if you desire) and it would have methods for the actions of the dog. You could have a void bark method that prints out "bark", a method that prints out "the dog is eating", and many more. Here is an example of a dog class:

public class Dog {

    private String name;
    private String breed;
    private int age;

    public String getName() {
        return name;
    }

    public void setName(String n) {
        name = n;
    }

    public String getBreed() {
        return breed;
    }

    public void setBreed(String b) {
        breed = b;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int a) {
        age = a;
    }

    public void bark() {
        System.out.println("Bark Bark");
    }

    public void eat() {
        System.out.println("Dog is eating");
    }

    public String toString() {
        return name + " is a " + breed + " that is " + age + " years old.";
    }
}

This is the main class of the program that has a Dog object in it.

public class DogArea {
    public static void main(String[] args) {
        Dog buddy = new Dog();
        buddy.setName("Buddy");
        buddy.setBreed("Golden Retriever");
        buddy.setAge(7);
        System.out.println(buddy.toString());
        boolean isAngry = true;
        if (isAngry) {
            buddy.bark();
        }
    }
}

So you would have 2 files in your project folder named: Dog.java and DogArea.java. To run your program, you would just compile the DogArea.java and then run it. Because this file contains the main method, the compiler will compile this class and all the other necessary classes.

Constructors

Notice how in the code above, an object was created and then its variables were set using the methods within the Dog class. This is okay, but what if you wanted to just set all the variables when the object was first created? That is what constructors are for.

A constructor is a special method that is called automatically when an object is created, and it can't return anything. You can use constructors to print info to the console when an object is created, set the vales of the object's variables, or just about anything else like a normal method (except return a value). You have actually already used constructors before. Whenever you make a scanner object, the last part of the code is new Scanner(System.in). The System.in is actually an argument or parameter that goes into the constructor when you create the object.

Making a Constructor

To make a constructor, you simply make a method that is the same name as your class and has no return type modifier. For example, the Dog class could have a constructor that looks like this:

public Dog(String n, String b, int a) {
    name = n;
    breed = b;
    age = a;
    System.out.println("A new dog named " + name + " was born".);
}

In your DogArea class, you can now make a new Dog object and set its values with just one line like this Dog buddy = new Dog("Buddy", "Golden Retriever", 0);. This way the user of your class or you won't have to call the set methods later. The variables that you pass into the constructor are separated by commas and do need to be in order. If you don't pass them in order then the compiler will see this and give you an error. Also, the constructor will give a quick print out letting the user know a dog was created.

Field Variables

If the values of a variable are not private, you can access the variables of an object directly by typing objectName.variableName. This is not recommended though because you should make your variables private so the user can't do something illogical with the variable like set the value of a dogs age to a negative number.

This is why it is always recommended to make public “get” and “set” methods that will return the values of variables and set the values of variables (if you want to allow your user to do that) in a safe way that you preprogrammed.

If you don't want a user having any access to your variables, then make them private and don't create any get or set methods.

Static

A static is a keyword that means that whatever you apply the static keyword to is connected to the class itself and not an object. This means that you can call or utilize static elements within a class without first creating an object. You often use static elements for things that all of the objects for a class share, but don't have unique values for. Static can be a little tricky to grasp initially without examples, so these should help.

A static method is a method that can be called from a class without first creating an object. In the methods tutorial, you were told to make methods static when methods were initially introduced to you. This is because the methods you were making were being called without using an object. The System.out.println() method is static because you never made an object to call it from. Furthermore, if you made a Dog object called spot and said spot.getAge() this would not be a static method because it is connected to your specific dog object. Static methods are called by first specifying the class name and then the method with a dot in between. If you were making a special math class called MyMath, you might want to make a method that returns the value of Pi. Because Pi will always be the same number and it will not be specific to an object, you should make this a static method, and would call it like this MyMath.getPi().

You can also have static variables that belong to the class and not a specific object. A variable like breed would not be static in the Dog class because breed is specific to each object. A good example of a static variable would be making a static int called numberOfDogs. Each time a dog object is created, you can add one to this variable, so it will count the number of dogs that will be created with this Dog class. If you did not make numberOfDogs a static variable, then this would be impossible from the class. You could then make a static method that returns the numberOfDogs, so the use will know how many dogs have been created from the Dog class. Here is an example of a dog class that uses static elements.

public class Dog {
    private String name;
    private String breed;
    private int age;

    private static int numberOfDogs;

    public Dog(String n, String b, int a) {
    name = n;
    breed = b;
    age = a;
    System.out.println("A new dog named " + name + " was born".);
    numberOfDogs++;
    }

    public int getNumberDogs() {
        return numberOfDogs;
    }

    public String getName() {
        return name;
    }

    public void setName(String n) {
        name = n;
    }

    public String getBreed() {
        return breed;
    }

    public void setBreed(String b) {
        breed = b;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int a) {
        age = a;
    }

    public void bark() {
        System.out.println("Bark Bark");
    }

    public void eat() {
        System.out.println("Dog is eating");
    }

    public String toString() {
        return name + " is a " + breed + " that is " + age + " years old.";
    }
}

}

This is mostly the same code as before but now a static int has been made that represents the number of dogs. Each time a dog is created, the constructor adds one to the static variable. There is also a static method that will return the number of dogs.

this keyword

The this keyword is used when you are referring to elements of the object itself within the class. For example, let's say your code looked like this

public class Dog {
    private String name;

    public Dog(String name) {
        name = name;
    }

    public String getName() {
        return name;
    }
}

Let's say you make an object from this class and use getName() to print out the name you passed in. getName() will return a blank string. Can you figure out why?

In the constructor you have a variable called name and then the field variable is also called name, so there are some naming issues. As said in the earlier tutorials, the method will always use the more local variable; therefore, in this case the code is just creating a variable called name within the constructor and setting it equal to itself. So the constructor essentially does nothing. You would fix this by using the this keyword:

public class Dog {
    private String name;

    public Dog(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}

The code in this example will work as intended because this lets the JVM know that you are referring to the global variable that belongs to the object itself, and not this local method. Also, the this keyword is really only necessary if your local variable names are the same as your global variable names, so you could avoid using the this keyword if you just don't repeat global variable names. A lot of this is dependent on the style that your professor or employer prefers.

toString method

Have you ever tried to print out an object like this System.out.println(ObjectName)? If you have then you most likely got a weird sequence of characters that looks something like this User@18fd984. This is because the console is printing out the memory address of the object. The memory address just tells you where the object is stored in the system essentially. What if you wanted your own custom message to be printed when you printed an object? This is possible with the toString() method.

The toString method has a general form like this public String toString() {return "Your String";} . So if you try to print out the object itself, then this message will be called automatically. It is a special feature within Java.

Exercise

This will be a larger exercise than normal, but it will be good practice for what you have learned so far. Your mission, should you choose to accept it, is to make a class called Movie that has a String for the title, a String for the director, and an int for the year. You should make a constructor that sets all of these values, and get methods for all of them as well. Then make a Test class that creates some movies (any movies you want), and prints out the name and year of each movie. Try to do this with as little code as possible. (Hint: you might want to make a method that prints the info of the movie for you)

Movie Class

public class Movie {
    private String title;
    private String director;
    private int year;
    public Movie(String t, String d, int y) {
        title = t;
        director = d;
        year = y;
    }

    public String getTitle() {
        return title;
    }

    public String getDirector() {
        return director;
    }

    public int getYear() {
        return year;
    }

    public void setTitle(String t) {
        title = t;
    }

    public void setDirector(String d) {
        director = d;
    }

    public void setYear(int y) {
        year = y;
    }

    public String toString() {
        return title + " (" + year + ")";
    }
}

Test Class

public class Test {
    public static void main(String[] args) {
        Movie m1 = new Movie("Shawshank Redemption", "Frank Darabont", 1994);
        Movie m2 = new Movie("Interstellar", "Christopher Nolan", 2014);
        Movie m3 = new Movie("2001: A Space Odyssey", "Stanley Kubrick", 1968);
    }
}

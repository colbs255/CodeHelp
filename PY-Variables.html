Variables

What is a variable?

You essentially store all information in variables. You can store a number, name, age, color, or just about anything in a variable. Understanding variables is very important for programming in general.

Types of Variables

The basic data types in Python are Numbers, Strings, and Booleans. There are several types of numbers including a integer, float, and long, but the main ones you would use are the float and integer. There are also compound data types that just hold a series of these basic data types which you will learn soon.

Variables Syntax

Declaring variables in Python generally has this syntax variableName = variableValue. Whenever you want to create a new variable, you just give it a name and assign a value to it. Python is different from many languages because it does not require the programmer to declare the type of variable like in Java or C++. This means that the Python interpreter actually infers the type of variable based on the data you assign to it. After instantiating a variable, you can later assign it a new value by typing variableName = newValue. These concepts apply to all types of values, so make sure you learn them. Here are some examples:

  • duncansAge = 18
  • battingAverage = .192
  • name = “Andy”
  • ronIsBest = True

Also, remember that variable names are case sensitive. So duncansAge is not the same as DuncansAge.

int

You can probably infer that int is short for integer which is just a whole number (i.e. -1, 0, 1, 2, etc..). They are good for representing simple whole numbers. In many languages, if you try to divide an int by an int, the answer will be essentially truncated (integer division), python 3 is a litte different in that doing jimsAge = 9/4 will indeed set jimsAge to 2.25 instead of 2. If you want to use integer division then use the // operator.

float

A “float” is like an int, but it can contain a decimal and can hold very large numbers. If you are dealing with numbers that can have a chance of not being whole, you should use a float. In practice, you mostly won’t have to worry about the distinction between floats and ints because Python will infer the type you want for you.

boolean

A “boolean” can either be set to True or False, nothing else. Boolean variables are often used to make code more readable. For example, you can type hasGlasses = True. Later on we will discuss “if statements” which will do something only if a condition is True, and Booleans are often used in these if statements. You can also set Booleans equal to expressions like this isGreater = (x > 5). This statement will set the boolean variable to True if x is greater than 5. If it is not greater, then the variable will be set to False.

Also, notice that True and False do not use quotation marks (because that would make the data type a string) and they are capitalized.

string

A string can be used to represent a word, a phrase, a sentence, or even a paragraph, and is denoted with quotation marks. So “11”, “bird”, “False” are all strings but 11, 23, and False are not strings. This is a simple but important distinction that you should know in programming.

What can you do with variables?

You can do a lot with primitive variables, including adding, subtracting, multiplying, or dividing. Take a look at this example

radius = 4
pi = 3.1415
circleArea = pi * radius * radius
print(“Circle area:”, circleArea)
	
x = (5/2) + 1
print(x)

In the first section an int variable called radius was initialized and equal to 4. Then a float called pi was initialized (float because it is a decimal). circleArea is interesting because it was set equal to a mathematical expression. circleArea in this case is a float as well because a float was used in the mathematical expression.

In the next section, an int variable called x was created, and set equal to a mathematical expression. The (5/2) part will return 2.5 not 2, and then add 1 to get 3.5.

Lastly, notice how a , was used inside of the print statement. The comma essentially tells the interpreter to add a space to the output and then print the next variable which is circleArea in this case.

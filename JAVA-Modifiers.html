Modifiers

Modifier words are the special keywords that are typed before methods, variables, and classes. These include words like: public, private, final, and static. In this lesson, you’ll learn what all of these mean.

Access Modifiers

Public and private are access modifiers. Access modifiers determine what classes can access these variables or methods. By default, methods and variables are visible by other classes that belong in the same package (packaging is just a way of organizing Java classes). Public means that all classes no matter their package can access your method or variable. Private means that the method or variable can only be accessed by things within the same class. Generally, variables within methods can be set to default (no access modifier), global variables should be private, and methods should be public. This is so people can’t interfere with the variables of your class unless they use one of the methods that you specifically designed. There is also another access modifier called “protected” that isn’t used as often. Protected is the same as default except all classes that extend the class can access the variable, even if the subclass is from a different package. We will get into inheritance soon, so don’t worry if you didn’t understand that.

Final

The final keyword means the variable can’t be changed. If you declare a variable as final, and try to change its value later then an error will occur.

Static

Static means something belongs to the class itself, and you do not use objects to access them. The methods in your classes have been static so far because you have not learned how to make objects yet, which you will soon.

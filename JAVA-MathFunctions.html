Math Functions

Java has a lot of built in math tools that make programming easy and more intuitive. The first thing you should know is that all of these tools belong to the Math class. Fortunately, you do not have to import the math class, all you have to do is type Math.functionYouWant(Parameter). There is a cheat sheet at the bottom of the page that shows what all of the main Math functions do.

Math.abs()

The absolute value method (a method is essentially a function) takes in any kind of number (int or double), and returns the absolute value of it (the numbers distance from 0). So Math.abs(-3.2) would return 3.2.

Math.pow()

This takes 2 arguments. The first is the base, and the second is the exponent. For example, Math.pow(2, 3) would return 8 because 2 to the third power is 8.

Math.floor() and Math.ceil()

Math.floor() takes in a number and rounds it down to the nearest whole number or integer. Math.ceil() does the opposite. It rounds up to the nearest integer. Math.floor(2.8) will return 2 and Math.ceil(2.3) will return 3.

Trigonometry functions

There are several trigonometry functions, they all take in radians and return radians not degrees with the exception of toRadians() and toDegrees. There is a Math.cos(), Math.sin(), and a Math.tan() function. All of these take in an angle in radians and return the result in radians. The Math.toRadians() method takes in an angle in degrees and returns the equivalent angle in radians and the Math.toDegrees() does the opposite.

Math.min() and Math.max()

The Math.min() method takes in 2 different numbers separated by commas and then returns the lower number of the two. The Math.max() takes in 2 numbers as well and returns the greater of the 2.

Cheat Sheet

public class CheatSheet {
    public static void main(String[] args) {
        Math.abs(-10.1); // returns 10.1 absValue of -10.1 is 10.1
        Math.pow(4, 2); // returns 16 because 4^2 = 16

        Math.floor(4.7); // returns 4 because 4.7 rounds down to 4
        Math.ceil(4.3); // returns 5 because 4.3 rounds up to 5

        Math.sin(0); // returns 0 because sin of 0 radians is 0
        Math.cos(0); // returns 1 because cos of 0 radians is 1
        Math.tan(0); // returns 0 because tan of 0 radians is 0
        Math.toDegrees(3.14); // returns ~180 because pi radians is 180 degrees

        Math.min(3, 5.2); // returns 3 because 3 is less than 5.2
        Math.max(3, 5.2); // returns 5.2 because 5.2 is greater than
    }
}

Assignment Shortcuts

You will often change the value of variables while programming. Let’s say you just wanted to add the number 1 to your variable. One way is like this varName = VarName + 1;. This is valid in Java because it evaluates the expression before assigning it to the variable. This code gets the value of the variable, adds 1 to the variable, and then assigns the variable that new value. A shortcut that you will see often in java to doing this is varName++;. This is a shortcut that will add 1 to your variable. It is 100% equivalent to varName = VarName + 1;; it is just a nice feature that the creators of Java added.

Let’s say you wanted to add a number that isn’t necessarily 1 to your variable. There is a shortcut for that also that looks like this varName += 2, you can use this for whatever number you want.

These shortcuts also apply for subtraction. You can do varName--, which will subtract 1 from the variable and varName -= 2 which will subtract 2.

There is also a shortcut for multiplication that looks like this varName *= anyNumber. This will multiply your variable by the number you enter and set it equal to that new value in the process.

One more thing to note is that ++varName and varName++ are actually both valid actions in Java, but they work differently (this distinction is commonly tested in college CS courses and interviews). The distinction can be seen when you pass them into the println method. Let’s say you have an int called myNumber that equals 4. If you typed System.out.println(myNumber++);, what will be printed out? 4 or 5 because the code is adding 1 to myNumber? This code will print 4, because myNumber++ will return the myNumber value before adding 1 to it. What if you typed System.out.println(++myNumber);? This would print 5, because ++myNumber will add 1 to the variable and then return it. So make sure you know the difference between these 2 forms and know when you should use them.

Exercise

Hopefully this gives you an idea of why many scientists and mathematicians use programming languages and how you can use them too. Try to make a program that asks for two angle measures in degrees, converts the higher angle measurement to radians, and then prints the absolute value of the cosine of the number. Yes, this might be difficult for you (there are many different ways to do it), but it will give you a better idea of how methods work in Java.

Solution 1

import java.util.Scanner;
public class CrazyMath {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        System.out.println("Enter angle 1:");
        double angle1 = input.nextDouble();

        System.out.println("Enter angle 2:");
        double angle2 = input.nextDouble();

        double higherAngle = Math.max(angle1, angle2);
        double inRadians = Math.toRadians(higherAngle);
        double cos = Math.cos(inRadians);
        double absVal = Math.abs(cos);

        System.out.println("Answer: " + absVal);
    }
}

This is a perfectly valid solution, but look at this solution.

Solution 2

import java.util.Scanner;
public class CrazyMath {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        System.out.println("Enter angle 1:");
        double angle1 = input.nextDouble();

        System.out.println("Enter angle 2:");
        double angle2 = input.nextDouble();

        System.out.println("Answer: " + Math.abs(Math.cos(Math.toRadians(Math.max(angle1, angle2)))));
    }
}

This solution is a bit extreme, but it demonstrates how functions can be called within each other and required less code. Take a moment to conceptualize what exactly is happening. Start from the middle and then work your way out.

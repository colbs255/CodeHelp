Control Statements

You have already learned how to command the computer, and now you will learn how to let the computer make decisions for itself with control statements. It isn’t artificial intelligence, but it’s still pretty cool, and you will use these in just about every program you write.

if statement

The if statement works the way you would expect. It says if this condition is true, then do this. Here’s what an if statement generally looks like.

if (yourCondition) {
    //do the code inside the curly brackets
}

Here's an example

int x = 5;
if (x >= 5) {
    System.out.println("X is greater than or equal to 5");
}

The computer looks inside of the parentheses and evaluates if x is greater than or equal to 5. Since x is equal to 5, the program will print “X is greater than or equal to 5”.

The if statement also works great with booleans. Heres an example

boolean hasGlasses = false;
if (hasGlasses) {
    System.out.println("Martha has glasses");
}

The statement will evaluate if the boolean hasGlasses is set to true. Since the statement is set to false, the code inside the curly brackets will not do anything.

if and else

a) In the case that your test condition is not true, you can use an else statement to do something “else”. For example,

int x = 5;
if (x == 4) {
    System.out.println("X is equal to 4");
} else {
    System.out.println("I don't know what x is... but I know it's not 4.");
}

The == operator tests if 2 variables are equal. Whenever you want to see if something is equal, use ==. DO NOT USE =. The = is the assignment operator. It is only used to set something equal to something else. This program checks if x is equal to 4. Since it does not equal 4, it will run the code in the else statement.

else if

if and else statements are great if you only have 2 outcomes, but if you have more than one outcome then you’ll need an “else if” statement. You essentially chain else if statements onto an if statement to check several conditions so your code can respond to different possibilities. Here is an example:

int age = 19;
if (age <= 3) {
    System.out.println("You are a toddler");
} else if (age <= 12) {
    System.out.println("You are a kid");
} else if (age <= 17) {
    System.out.println("You are a teen");
} else {
    System.out.println("You are an adult.");
}

The code will start with if statement and if true, then it will run the code inside the first curly brackets. If it is not true, then it will move on to the next condition and will repeat this process until it either reaches a condition that is true or it reaches the else statement. In this case, the code in the else statement will execute since 19 is not true for any of the conditions

Switch Statements

Switch statements are roughly the same as else if statements in a different format. They are often used when you provide a user with a list of options to pick from. Here is an example:

Scanner input = new Scanner(System.in);
System.out.println("1. Iron Man");
System.out.println("2. Thor");
System.out.println("3. Hulk");
System.out.println("4. Black Panther");
System.out.println("Which character would you like to play as? Enter a number.");
int choice = input.nextInt();

switch (choice) {
    case 1: System.out.println("You are Iron Man");
    break;
    case 2: System.out.println("You are Thor");
    break;
    case 3: System.out.println("You are Hulk");
    break;
    case 4: System.out.println("You are Black Panther");
    break;
    default: System.out.println("You did not pick a valid choice.");
    System.out.printnl("You were defaulted to Ant Man");
    break;
}

In this code, the user is given a list of super heroes to play as and must enter a number to pick. The switch statement takes in this variable and matches it with the numbers in the following statements. If none of the numbers match, then the default case will run. You must put break at the end of each case, and you can put multiple statements in each case.

&& and || operators

Let’s say you only want a block of code to run when 2 conditions are true. You might be tempted to write something like this

boolean canFly = true;
boolean hasSuperStrength = true;
if (canFly) {
    if (hasSuperStrength) {
    System.out.println("You are superman.");
    }
}

a) This code is totally valid, but it is not easy to read, and there is a much simpler way. Take a look at this:

boolean canFly = true;
boolean hasSuperStrength = true;
if (canFly && hasSuperSpeed) {
    System.out.println("You are superman.");
}

This is easier to read, easier to write, easier for the machine to run, and essentially equivalent to what was coded above. The && is the and operator, so if we translate this code to English, it would be like: “If canFly is true and if and only if hasSuperStrength is also true, then print "You are superman." Remember that the block of code will only be called if both of these conditions are true. If one of them is false, then the following code will be skipped.

Let’s say you want a block of code to run if either one of 2 conditions are true. You might want to write this:

boolean canFly = true;
boolean hasSuperStrength = true;
if (canFly) {
    System.out.println("You have super powers.");
}
if (hasSuperStrength) {
    System.out.println("You have super powers.");
}

This code is alright, but if both of the variables are true then the same statement will be printed out twice. Here is another trick:

if (canFly || hasSuperStrength) {
    System.out.println("You have super powers.");
}

Once again, this is simpler and better. The || is the or operator, so in English this code says “If canFly is true or if hasSuperStrength is true, then you have super powers.” It will also print the statement at most one time, so there is no redundancy.

If you ever find yourself writing a bunch of if statements or if statements nested inside of other if statements. Think about using one of these operators.

if vs else if vs switch

It is important that you know the differences between each of these so you know when to use them.

Why would you use an else if when you could just use multiple if statements?

Answer: When you create an else if chain, code will only run in one of those else if segments. If you did multiple if statements then there’s a possibility that all of those statements can be true and run their respective code, so you need to figure out if you only want one thing to happen or potentially more.

Why use a switch statement instead of an else if chain?

Answer: Switch statements are good for when the user has relatively few choices and you want something to happen only when the user’s choice equals something specific. Else if chains are good when for when the user has many options and you want the code to run when the user picks a choice in a certain range.

Challenge

Write a program that will print what kind of shape something is based off of these booleans: has4Sides, allSidesEqualLength, and allCorners90Degrees. The possible shapes are square, rectangle, other quadrilateral, or not a quadrilateral. Only print one of these in your option, not more than one. Hint: A square has 4 sides, all sides are equal length, and all corners are 90 degrees. A rectangle has 4 sides and all corners are 90 degrees. A shape is a quadrilateral if it has 4 sides.

Answer

if (has4Sides && allSidesEqualLength && allConers90Degrees) {
    System.out.println("Square");
} else if (has4Sides && allCorners90Degrees) {
    System.out.println("Rectangle");
} else if (has4Sides) {
    System.out.println("Other Quadrillateral");
} else {
    System.out.println("Not a quadralliteral");
}

There are many ways to do this, but this is one of the simplest ways. Notice how the code tests for the most specific case first, and then tests for more general cases as it goes down; you generally want to use this strategy for efficient code.

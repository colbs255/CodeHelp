Variables

What is a variable?

You essentially store all information in variables. You can store a number, name, age, color, or just about anything in a variable. Understanding variables is very important for programming in general.

Types of Variables

Java essentially has 2 main types of variables: primitive variables and reference variables. Reference variables are special variables that “reference” objects, and we will get to objects later. For now, we will focus on primitives (objects are essentially a special collection of primitives). Primitive variables are basic variables stored by bits, and Java has eight primitive variables: boolean, byte, char, short, int, long, float, and double. Don’t worry, you will really only use boolean, char, int and double though. The others are generally not very useful compared to these main 4.

Variable Syntax

Declaring a variable generally has this syntax variableType variableName = variableValue;. For example int billsAge = 21; . This statement essentially says make a variable of the type int (short for integer) called billsAge and set its value equal to 21. Whenever you make a new variable, you have to tell Java its type and assign it a name, but you do not have to immediately assign it a value. For example, you can do int billsAge; and then on another line type billsAge = 21;. This is equivalent to the previous code example. When you name a variable and say its type, this is called declaring a variable, and when you give it a value for the first time, this is called instantiating a variable. After instantiating a variable, you can later assign it a new value by typing variableName = newValue. For example, billsAge = 22. These concepts apply to all types of values, so make sure you learn them. Here are some examples.

  • int maggiesAge = 21;
  • double battingAverage = .192;
  • char favoriteLetter = 'c';
  • boolean hasWon = false;
  • hasWon = true;

Also, remember that variable names are case sensitive. So maggiesAge is not the same as MaggiesAge.

int

“int” short for integer, is number that can never be a decimal (i.e. -1, 0, 1, 2, etc..) and can hold values from -2^31 to 2^31 - 1 (-2,147,483,648 to 2,147,483,647). If you try to make an int a decimal you will get errors in your code. They are good for representing simple whole numbers. If you try to divide an int by an int, the answer will be truncated. For example, if you do int jimsAge = 9/4;, jimsAge will be set to 2 not 2.25! Try to be mindful of this when programming.

double

A “double” is like an int, but it can contain a decimal and can hold very large numbers. If you are dealing with numbers that can have a chance of not being whole, you should use a double. Double values will not truncate numbers like ints. For example, if you do double jimsAge = 9/4, jimsAge will be set to 2.25 not 2.

boolean

A “boolean” can either be set to true or false, nothing else. Boolean variables are often used to make code more readable. For example, you can type boolean hasGlasses = true;. Later on we will discuss “if statements” which will do something only if a condition is true, and Booleans are often used in these if statements. For example, if (hasGlasses) System.out.println(“The person has glasses”);. This statement will only print if the variable is set to true. You can also set Booleans equal to expressions like this boolean isGreater = (x > 5);. This statement will set the boolean variable to true if x is greater than 5. If it is not greater, then the variable will be set to false.

Also, notice that true and false do not use quotation marks and are not capitalized. This is because these are special words reserved for the Java language, so never set a boolean equal to “true” or “false” or True or False. You will only use true or false.

char

A “char” is just a character. You can set a char equal to just about any character including a number, symbol, or letter. Here is an example of a char declaration and instantiation: char favoriteLetter = ‘m’;. Notice that you wrap chars in single quotes, so do not forget them. While you might not use chars very often while programming, they are important to understand for the next variable we are about to discuss, Strings.

String

A string is essentially a collection of chars. They can be used to represent a word, a phrase, a sentence, or even a paragraph. String variables are technically not primitives, they are reference variables that represent objects. However, Java essentially treats Strings them as primitives, and you will learn more about this later. For now, just know what a string is and how you use them like this: String myName = “Steve”;. Notice that String is capitalized and it always has to be for your code to work.

What can you do with variables?

You can do a lot with primitive variables, including adding, subtracting, multiplying, or dividing. Take a look at this example

public class SampleVariables {
    public static void main(String[] args) {
        int radius = 4;
        double pi = 3.1415;
        double circleArea = pi * radius * radius;
        System.out.println("Circle Area: " + circleArea);

        int x = (5/2) + 1;
        System.out.println(x);
    }
}

In the first section, an int variable called radius was initialiezed and set equal to 4. Then a double called pi and another double called circleArea were initialized. Notice how this variable was set equal to a mathematical expression. The code calculates the expression and then sets the variable equal to that value (it does use standard order of operations.

In the second section, an int variable called x was created. You might think that the expression would divide 5 by 2 to get 2.5 and then add 1 to get 3.5, setting x equal to 3.5. It will not do this though because the 2 and the 5 in the expression are ints (put a .0 at the end to make them doubles). The code will essentially truncate (or cut off the decimal) 5/2 to 2 and then add 1 to get 3. This is often called “integer division".

Lastly, notice how a + was used inside of the println statement. The + essentially says take the string and add something else to the end of it to make a new string.

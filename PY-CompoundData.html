Compound Data Types

A compound data structure is essentially a special variable that can hold multiple basic data types like ints, strings, or floats. The main ones in Python are lists, tuples, and dictionaries.

List

Let’s say you are Usain Bolt and you want to record all your 100m dash times. You can do this with a list. The general syntax for a list is

listName = [val1, val2, val3]

To record your 100m dash times, you can make a list with your 100m dash time values like this:

boltsTimes  = [9.58, 9.71, 9.62, 9.68]

Each value in the list can be accessed with an index. Remember this, programs usually start with 0 rather than 1, so the first element in a list actually has the index 0, the second has 1, and the third has 2. The syntax for accessing an element in a list is listName[index], so if you wanted to use the first element in this example you would type boltsTimes[0], meaning you could print the first value in the list by typing

print(boltsTimes[0])

Another key thing to note is that lists can hold variables of different types. So you could make a list holding a string, an int, and a float like this. list = [“Maggie”, 21, 2.34].

Tuple

A tuple works the same way that a list does, but has a couple minor differences. The first is that tuples are immutable, meaning you can’t change the values of a tuple or change its size once it is created, but you can with a list. They are different syntactically because they use parentheses instead of brackets. Here is what they look like:

tupleName = (val1, val2, val3)

So the boltsTimes example would look like this as a tuple

boltsTmes = (9.58, 9.71, 9.62, 9.68)

Unlike lists, you would get an error if you tried to change the value of an element in a tuple like this

boltsTimes[0] = 9.59

Use a tuple or a list?

Tuples should be used if you don’t plan to change the values of the variable after initializing the variable because the tuple has slightly better performance. If you have to change any data later then you should list. Most of the time you will probably be using a list.

Dictionary

Dictionaries are similar to lists, but each element is indexed by unique keys rather than just a range of numbers starting with 0. Think of a dictionary as a list of key and value pairs. Here is the general syntax

dictName = {key1 : val1, key2 : val2, key3 : val3}

Here is an example of dictionary that maps peoples first names to their ages

namesAges = {“Marky” :  20, “Ricky” : 21, “Danny” : 23, “Terry” : 25, “Mikey” : 24, “Davey” : 25, “Timmy” : 26, “Tommy” : 23, “Joey” : 24, “Robby” : 22, “Johnny” : 24, “Brian” : 26}

You can now print the age of Joey by typing:

print(namesAges[“Joey”)

Notice how keys can have the same values, but no 2 values can have the same key. Every key in dictionary must be unique. If you typed namesAges[“Joey”] = 27. The value that corresponds to the Joey key will be changed; it would not add a new pair. You can add a new pair, however, if you do the same thing with a unique key.

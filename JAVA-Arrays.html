Arrays

What is an array?

An array is an indexed collection of variables of the same type

Why would you use an array?

Let’s say you are a teacher and want to find the class average on your last test. You would probably enter in all of your test scores as variables like double s1 = 89; double s2 = 92; double s3 = 84; and repeat this for all students. Then you would probably find the average like this: int average = (s1 + s2 + s3 + s4 + s5…..)/(numberOfStudents); . This could get very tedious, and fortunately, there is a much faster method that utilizes arrays

Array Syntax

Arrays can be created in several ways

//General case
varType[] arrayName = new varType[numberOfElements];

//Example
double[] testScores = new double[10];

This will create an array that can hold a maximum of 10 doubles. Java knows this is an array because of the brackets after the variable type. You can also make an array like this double testScores[] = new double[10];, with the brackets at the end of the variable name, but usually it is done the first way.

After initializing the array, you can set the values of the variables within the array like this:

//General case
arrayName[indexOfElement] = value;

//Example
testScores[0] = 89;
testScores[1] = 92;
testScores[2] = 84;

Remember, arrays start with 0, not 1. Java beginners often forget this, leading to errors in their code, so remember that. So technically the 1st element in the array has an index of 0 like in the example above.

Java has a shortcut for declaring an array and assigning values to the elements in the array like this.

General Case
varType varName = [value0, value1, value2, value3];
//Example
String studentNames = ["Steve", "Bill", "Mark"];

This way you do not have to waste your time writing code like this studentNames[x] = “name”; repeatedly.

Key Point: Arrays can be made of any variable type (String, int, double, char, etc), but an array can only hold 1 type (you can’t have a string in one element and a double in another element of an array).

Array Defaults

When you create an int array or a double array like this int[] numbers = new int[5]. Each element will be assigned a value of 0 (0.0 if it’s a double) until you change the value.

If you do the same with a string array, each element will automatically be assigned a value of “null” until you change the value. It won’t just be a blank string

For a boolean array, each element will automatically be assigned a value of false until you change the value

For a char array, each element will be assigned the null character ‘u/0000’.

Array Length

You can find the number of elements within in array by typing arrayName.length. So if your array had 5 elements then the length of your array will be 5

Arrays with loops (where arrays are really useful)

Arrays by themselves are nice for organizing data, but they are much more useful when you utilize them with loops (usually the for loop). Here is how the test average scenario at the beginning of the lesson could be programmed with an array and a simple for loop:

for(arrayType variableName: arrayName) {
    // do stuff with variable name
}

This code will essentially add one of the elements within the testScores array in each iteration of the for loop to the sum variable. x in the for loop starts at 0 and then the for loop runs as long as x is less than the length of the testScores array, so the for loop will have 3 iterations. Note how a < was used in the code and not <=. If <= was used then there would be an error because the code will be trying to access an element with an index of 3 in the code, and there are no elements with an index of 3. The highest index is 2 since there are 3 elements and the indexing starts at 0.

Enhanced for loop

Enhanced for loops are special for loops specifically designed for arrays. They are much cleaner and simpler, but they do have some limitations. The general syntax of a for loop looks like this:

int[] numbers = [23, 34, 35, 11, 10];
int sum = 0;
for(int x: numbers) {
    sum += x;
}

An enhanced for loop essentially makes a copy of the value of an element in each iteration of the loop. You can then use the value as if you were specifying that specific element of the array. Here is an example

double[] testScores = [89, 92, 84];
double sum = 0;
for (int x = 0; x < testScores.length; x++) {
    sum += testScores[x];
}
double average = sum/testScores.length;

In the example, an enhanced for loop is created using the numbers array. In each iteration of the for loop, a different copy of a value in the numbers array will be made and then utilized as x. X is essentially the same value as what numbers[x] would be in a normal for loop with x as the main variable

This is useful because the code is abbreviated and you do not have to worry about the length of the array; however, you do need to be careful with enhanced for loops because they do have some limitations. You can’t use an enhanced for loop to change values of elements in an array. For example, if you put x = 4 in the body of the for loop within the previous example, you might think that this would set all the elements in the for loop equal to 4, but it wouldn’t. This is because, x is actually a copy of the value of the element that is local to that for loop. X does not actually represent the elements of the for loop, so the code would essentially do nothing in the end.

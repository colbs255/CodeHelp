Hello World

Getting Started

You are about to write your first program. If you are using a text editor, then click File > New (cmd + N on mac or ctrl + N on windows) then File > Save (cmd + S on mac or ctrl + S on windows) and name it HelloWorld.py. You can save the program anywhere you want, but remember where you save it.

Your First Program

print(“Hello World”)
# prints hello world

Just type that… and that’s your program. Python is cool. If you’re coming from a language like Java then this might seem like magic to you because it is much simpler to make a basic hello world program in Python. If you are using an IDE, then just press the run button (should look like a play button) and move on to the next section. If you are using a text editor, then running the program is a little more complicated because you need to use terminal or command prompt. On a mac, press cmd + space and type “ter” then select the terminal application. Now you have to navigate to the directory where your file is located. Find the folder that your file exists it then type cd on the terminal and drag the folder into your terminal. This should put the path to your file into your terminal and look something like this cd ~/Users/username/Documents. You have just told the terminal to navigate to the directory your class is in. Now type python HelloWorld.py and your program should run. If you are on Windows, go to the “command prompt” and then do the same thing but replace the ls command with dir. Practice doing this! Being able to navigate using a terminal is an important skill. There are also ways to run pyhton programs directly from a text editor, so it would be a good idea to look into setting that up for your editor.

What does this code mean?

The code is using a function ( essentially method in Java) called print that takes in some sort of data and then prints it to the console. The data that you are printing in this case is called a string, which is essentially just normal text. Notice that there is a no semicolon or anything at the end of the statements, this is because Python actually uses lines to break up each line of code unlike Java that uses semicolons.

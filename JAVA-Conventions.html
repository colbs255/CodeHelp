Conventions

This is an easy tutorial that is concerned with style of code rather than function. You have probably noticed common conventions in these tutorials already like classes are always capitalized, variables are not, and everything is indented. These are common conventions in Java that make code easy to read and edit. This is very important for computer science jobs and even computer science classes (many professors take off points on assignments if these conventions are not followed). Some environments may prefer slightly different styles of code, but these are generally accepted in Java.

Naming

When making a variable final, it should be in all caps the words in the variable name should be separated with underscores like this: final int MY_FINAL_VARIABLE_NAME = 1;.

All other variable names (object names and primitive value names) should only have capital letters at the start of each word, but the first letter of the variable is always lowercase. This is called “camel case”. All of the variables in the sample codes of this site have used this convention.

The first letter of a class should always be capitalized.

Indentation

There is a debate over using tabs or spaces for indentation in programing. You might want to see what your professor or workplace prefers. I prefer to just set my IDE to automatically use 4 spaces whenever I tab (google how to do this on you IDE), and that seems to be the best of both worlds.

The header of your class should not be indented, but the methods and variables within your class should be indented. Whenever you put code inside of a structure like a loop, conditional statement, or a method you should indent the code. This makes the code much easier to read and edit. You can look at all of the sample code sections to see what this looks like.

Brackets

Generally, the opening bracket is on the same line as the code that the opening bracket “opens” like this:

for (int x = 0; x < 4; x++) { // the opening bracket is on the same line
    //this section is indented one more level
}
//see how this bracket lines up with the for part

You should line up the closing bracket with the beginning of the section the closing header corresponds to on separate line from the rest of the code. This makes it easier for the programmer to see which bracket corresponds to what.

if ((x * 2) >= 4) { //all of the operators have spaces and there is a space between the if keyword and the parantheses

}
